# Enfinity — Off-Road Semantic Segmentation

A semantic segmentation model for off-road environments built on a frozen **DINOv2-ViT-Large** backbone with a custom **Progressive Semantic Decoder Head** (`MultiLayerSegmentationHead`). The model classifies every pixel into one of 10 vegetation/terrain classes.

## Project Overview

| Item | Detail |
|---|---|
| **Task** | Pixel-wise semantic segmentation of off-road scenes |
| **Backbone** | DINOv2 ViT-Large/14 with registers (`dinov2_vitl14_reg`) — **frozen** |
| **Head** | `MultiLayerSegmentationHead` — 4-layer feature fusion + progressive 8× upsampler |
| **Classes** | 10 (Trees, Lush Bushes, Dry Grass, Dry Bushes, Ground Clutter, Flowers, Logs, Rocks, Landscape, Sky) |
| **Input resolution** | 952 × 532 px (nearest multiple of patch size 14 to 960 × 540) |
| **Best checkpoint** | `segmentation_head_best.pt` (included in repo) |
| **Reported Mean IoU** | **0.2995** on the provided test set |

## Environment & Dependency Requirements

### Prerequisites

- **Miniconda** or **Anaconda** (Python 3.10)
- **CUDA 11.8** compatible GPU (strongly recommended; CPU inference is very slow)
- Windows (batch scripts provided) or Linux/macOS (manual steps below)

### Automated Setup (Windows)

```bat
cd ENV_SETUP
setup_env.bat
```

`setup_env.bat` calls `create_env.bat` (creates the `EDU` conda environment with Python 3.10) then `install_packages.bat` (installs all dependencies).

### Manual Setup (Linux / macOS)

```bash
conda create --name EDU python=3.10 -y
conda activate EDU

# PyTorch + CUDA 11.8
conda install -c pytorch -c nvidia -c conda-forge \
    pytorch torchvision pytorch-cuda=11.8 -y

# Additional packages
pip install opencv-contrib-python tqdm

# DINOv2 backbone (loads via torch.hub from a local cache)
python -c "import torch; torch.hub.load('facebookresearch/dinov2', 'dinov2_vitl14_reg')"
```

> **Note:** The DINOv2 model is loaded from the local torch hub cache directory (`~/.cache/torch/hub/facebookresearch_dinov2_main`). The first run will download it automatically if it is not already cached.

## How to Reproduce Final Results

### Dataset layout expected

The scripts assume the following directory layout **one level above** the repo root:

```
../
├── Offroad_Segmentation_Training_Dataset/
│   └── Offroad_Segmentation_Training_Dataset/
│       ├── train/
│       │   ├── Color_Images/   # RGB images (.png)
│       │   └── Segmentation/   # Mask files (.png, same names)
│       └── val/
│           ├── Color_Images/
│           └── Segmentation/
└── Offroad_Segmentation_testImages/
    └── Offroad_Segmentation_testImages/
        ├── Color_Images/
        └── Segmentation/
```

### Reproducing the reported Mean IoU (0.2995)

```bash
conda activate EDU
cd /path/to/Enfinity

python test_segmentation.py \
    --model_path segmentation_head_best.pt \
    --data_dir ../Offroad_Segmentation_testImages/Offroad_Segmentation_testImages \
    --output_dir ./predictions \
    --num_samples 5
```

The script automatically picks up `segmentation_head_best.pt` if `--model_path` is omitted.

## Step-by-Step: Running the Model

### 1. Training

```bash
conda activate EDU
cd /path/to/Enfinity

python train_segmentation.py
```

**What it does:**
- Loads the frozen DINOv2 ViT-Large backbone from the local torch hub cache.
- Trains only the `MultiLayerSegmentationHead` (segmentation decoder).
- Uses **Focal Loss + Dice Loss** with pre-computed inverse-frequency class weights.
- Applies rich data augmentation (random flip, rotation, resized crop, color jitter, Gaussian blur, random erasing).
- Runs for **45 epochs** with AdamW (`lr=1e-4`, `weight_decay=1e-4`) and a 5-epoch linear warmup followed by cosine LR decay.
- Gradient accumulation (`accumulation_steps=4`) simulates a batch size of 4 while using batch size 1.
- **Auto-resumes** from the latest `checkpoints/epoch_*.pt` if interrupted.
- Saves the best checkpoint (by validation IoU) to `segmentation_head_best.pt`.
- Saves per-epoch training plots and metrics to `train_stats/`.

**Training outputs:**
```
checkpoints/epoch_<N>.pt        # Per-epoch checkpoint (resume support)
segmentation_head_best.pt       # Best model by Val IoU (used for inference)
train_stats/
    training_curves.png         # Loss & pixel-accuracy curves
    iou_curves.png              # IoU over epochs
    dice_curves.png             # Dice score over epochs
    all_metrics_curves.png      # Combined 4-panel plot
    evaluation_metrics.txt      # Per-epoch tabular summary
```

### 2. Testing / Inference

```bash
conda activate EDU
cd /path/to/Enfinity

python test_segmentation.py [OPTIONS]
```

| Argument | Default | Description |
|---|---|---|
| `--model_path` | `segmentation_head_best.pt` (auto-detected) | Path to trained model weights |
| `--data_dir` | `../Offroad_Segmentation_testImages/…` | Root of test dataset |
| `--output_dir` | `./predictions` | Where to save all outputs |
| `--batch_size` | `1` | Inference batch size |
| `--num_samples` | `5` | Number of side-by-side comparison images to save |

The script evaluates **all** images and saves predictions for every one of them.

### 3. Visualization Utility

`visualize.py` colorizes any folder of raw segmentation mask PNGs (single-channel, arbitrary integer values) into RGB images using random, consistent colors.

```bash
# Edit the input_folder variable at the top of visualize.py, then run:
python visualize.py
```

Colorized images are saved to `<input_folder>/colorized/`.

## Expected Outputs & How to Interpret Them

### Evaluation metrics (`predictions/evaluation_metrics.txt`)

```
EVALUATION RESULTS
==================================================
Mean IoU:          0.2995
==================================================

Per-Class IoU:
----------------------------------------
  Trees               : 0.3378
  Lush Bushes         : 0.0004
  Dry Grass           : 0.4536
  Dry Bushes          : 0.2273
  Ground Clutter      : 0.0000
  Flowers             : 0.0000
  Logs                : 0.0000
  Rocks               : 0.0569
  Landscape           : 0.6297
  Sky                 : 0.9716
```

- **Mean IoU** is the primary metric — higher is better (max 1.0).
- Classes with IoU = 0.0 are either absent in the test split or too rare for the model to learn reliably (Flowers, Logs, Ground Clutter).
- Sky achieves near-perfect IoU (0.97) as it is visually distinctive and well-represented.

### Per-class IoU bar chart (`predictions/per_class_metrics.png`)

A bar chart colored with each class's palette color. The red dashed line marks the Mean IoU. Classes with 0 IoU appear as empty bars.

### Prediction masks (`predictions/masks/`)

Single-channel PNG images where each pixel value is a class ID (0–9). These can be used downstream for further analysis or re-colorization.

### Colored prediction masks (`predictions/masks_color/`)

RGB images where each pixel is colored according to the class color palette. Suitable for visual inspection.

### Comparison images (`predictions/comparisons/`)

Side-by-side figures with three panels:
1. **Input Image** — the denormalized RGB input.
2. **Ground Truth** — the annotated segmentation mask in color.
3. **Prediction** — the model's output segmentation mask in color.

## Class Definitions & Color Palette

| ID | Class Name | Color (RGB) |
|---|---|---|
| 0 | Trees | `(34, 139, 34)` — Forest Green |
| 1 | Lush Bushes | `(0, 255, 0)` — Lime |
| 2 | Dry Grass | `(210, 180, 140)` — Tan |
| 3 | Dry Bushes | `(139, 90, 43)` — Brown |
| 4 | Ground Clutter | `(128, 128, 0)` — Olive |
| 5 | Flowers | `(255, 105, 180)` — Hot Pink |
| 6 | Logs | `(139, 69, 19)` — Saddle Brown |
| 7 | Rocks | `(128, 128, 128)` — Gray |
| 8 | Landscape | `(160, 82, 45)` — Sienna |
| 9 | Sky | `(135, 206, 235)` — Sky Blue |

> **Background / ignored pixels** are mapped to class ID `255` and excluded from all loss and metric calculations.
